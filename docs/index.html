FNLLA PHP is the maintained PHP application framework in the FNLLA product family. This docs set maps the repository contract, the starter-export workflow and the practical delivery rules for downstream websites and applications.

Version 1.0.11 keeps the framework intentionally explicit: request flow, UI runtime boundary and maintainer scripts are visible in the repository rather than hidden behind packaging layers.

Build real projects outside this repo

  • Framework mode: maintain fnlla/php, its scripts, docs and shared delivery foundations here.
  • Project mode: export a downstream starter with php fnlla make:project and build the real client application in that separate directory.
  • UI rule: keep FNLLA Web as the only supported UI runtime under public/vendor/fnlla-web/.

Keep these rules in your head

These are the habits that keep FNLLA PHP strong instead of slowly turning it into a one-project-only code dump.

Export before delivery work

Treat this repository as the framework source of truth. Use make:project to start real downstream websites and apps in their own repositories.

FNLLA Web stays mandatory

Do not replace the UI runtime with Tailwind, Bootstrap or another CSS framework. The shared layout and guard rails assume FNLLA Web is present.

MySQL is the official database target

The framework contract assumes PHP 8.3 with pdo_mysql and a reachable MySQL server for application persistence work.

Validation is part of maintenance

Run tests, lint, FNLLA Web validation, version-manifest checks and docs sync checks before calling a release state finished.
